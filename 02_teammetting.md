# 2차 프로젝트 정기 미팅 및 교수님 피드백 정리


---

## 1. 모델링 방향성 및 데이터 전략 피드백

### 1.1 피처 엔지니어링 (Feature Engineering)
- **현상황:** 초기 100개 내외의 피처 사용 시 노이즈 발생 가능성 존재.
- **수정 방향:** 단순히 피처를 줄이는 것이 아니라, 도메인 지식을 바탕으로 **유의미한 핵심 변수 10~30개**를 정교하게 선별하여 모델의 견고성(Robustness) 확보.

### 1.2 예측 스코프(Scope) 조정
- **현상황:** 2040년 장기 예측은 검증 수단이 부족함.
- **수정 방향:** 현실적으로 정책적 근거를 보장할 수 있는 **'2025년 기준 향후 3년 이내 폐교 위험군'** 탐색으로 모델 목적 구체화.

### 1.3 조건부 언더샘플링 (Stratified Undersampling)
- **방법:** 단순 무작위 샘플링이 아닌 **권역별(수도권 vs 비수도권)** 혹은 **행정구역별** 폐교/운영 학교 비율을 고려하여 언더샘플링 수행. 지역적 특성이 편향되지 않도록 데이터셋 밸런싱.

## 2. 검증 기법: 시계열 교차 검증 (Time-Series CV)

### 확장 윈도우 (Expanding Window) 방식 도입
데이터의 시간적 순서를 보존하기 위해 다음과 같이 학습 범위를 누적하며 검증 수행:
1. **Fold 1:** 전체의 70% 학습 → 다음 10% 예측
2. **Fold 2:** 전체의 80% 학습 → 다음 10% 예측
3. **Fold 3:** 전체의 90% 학습 → 마지막 10% 예측
4. **결과:** 각 Fold별 성능의 산술 평균을 최종 모델 성능으로 채택.

---

## 3. 도입 알고리즘: LightGBM 고도화

불균형 데이터셋 및 고차원 데이터 처리를 위해 LightGBM의 핵심 기능을 적극 활용함:
- **GOSS (Gradient-based One-Side Sampling):** 오차가 큰 데이터에 집중하여 학습 효율 및 정확도 향상.
- **EFB (Exclusive Feature Bundling):** 희소 특성을 묶어 처리 속도 최적화.
- **Categorical Support:** 범주형 변수(학교급 등)의 자동 처리를 통한 모델 안정성 확보.
