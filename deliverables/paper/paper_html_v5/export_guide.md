# Chrome 및 Edge 브라우저 기반 PDF 내보내기 가이드 (paper_html_v5)

본 문서는 `index.html` 파일을 A4 용지 기준 2페이지 분량의 학술 논문 레이아웃으로 완벽히 변환하여 PDF로 저장하기 위한 설정 방법을 기술합니다.

## 1. 기본 실행 방법

1. Google Chrome 또는 Microsoft Edge 브라우저를 실행합니다.
2. 단축키 `Ctrl + O`를 누르고, 생성된 `paper_html_v5/index.html` 파일을 선택하여 엽니다.
   * 브라우저 일반 화면에서는 웹 문서 특성상 단락들이 하나의 긴 페이지처럼 아래로 끝없이 연결되어 보입동작입니다. 이는 정상적인 브라우저 동작이며, 실제 페이지 분할 영역은 인쇄 미리보기에서 제어됩니다.

## 2. 인쇄 및 PDF 저장 설정 (Ctrl + P)

브라우저 화면에서 **`Ctrl + P`**를 입력하여 인쇄 설정 창을 엽니다. 최적의 레이아웃을 위해 아래의 세부 옵션을 설정합니다.

* **대상 (Destination)**: `PDF로 저장` (Save as PDF)
* **페이지 (Pages)**: `모두` (All) 또는 `1-2`
* **레이아웃 (Layout)**: `세로` (Portrait)
* **용지 크기 (Paper size)**: `A4`
* **여백 (Margins)**: `기본값` (Default) 또는 `사용자 설정` (Custom)
  * CSS 내부에서 이미 기획된 여백(`@page { margin: 15mm 14mm; }`)을 제어하므로 기본값 사용이 권장됩니다.
* **배율 (Scale)**: `사용자 지정` (Custom) $\rightarrow$ **`90% ~ 100%`** 범위 설정
  * v5 버전에서는 2페이지 하단 여백을 알맞게 채우기 위해 웹 시나리오, 결론 한계, 실무 유의점 문단이 보강되어 텍스트 총량이 증가하였습니다.
  * 따라서 배율을 **`90% ~ 100%`** 범위로 설정했을 때 가장 가독성이 높고 여백이 꽉 찬 2페이지 논문 레이아웃을 생성할 수 있습니다.
* **옵션 (Options)**: **`배경 그래픽` (Background graphics)** 항목 체크 활성화
  * 이 옵션을 활성화해야 초록(Abstract)의 옅은 회색 배경 박스 및 웹 강조 주의 단락의 색상 띠가 PDF에 올바르게 반영됩니다.
  * 머리글과 바닥글(Headers and footers)은 해제하는 것을 권장합니다 (인쇄 시 날짜 및 파일 경로가 인쇄되는 것을 방지).

## 3. 분량 조절 및 레이아웃 제어 트러블슈팅

텍스트 추가로 인해 인쇄 미리보기 시 분량이 어긋날 경우 아래 기준으로 대응합니다.

* **인쇄 결과물이 3페이지로 넘어가는 경우**:
  - 인쇄 미리보기 배율 설정을 **90% ~ 95%**로 조금 낮춥니다.
  - 여전히 3페이지로 넘어간다면, `index.html`의 5장 결론(References 직전)에 있는 **선택 문단(실무 적용 시 유의점)** 전체를 코드에서 삭제합니다.
* **2페이지 하단 여백이 여전히 과도하게 많이 남는 경우**:
  - 인쇄 배율 설정을 **95% ~ 100%**로 올려봅니다.
  - `index.html`에 **선택 문단(실무 적용 시 유의점)**이 정상적으로 포함되어 있는지 소스코드를 재확인합니다.
