# 🏫 전국 학교 통폐합 위험 예측 및 정책 시뮬레이션 프로젝트 분석 리포트

## 1. 프로젝트 개요 및 아키텍처 평가

본 프로젝트는 단순한 '폐교 예측'을 넘어 학령인구 감소, 상권 취약성, 학교 고립도 등의 맥락을 종합하여 **'통폐합 검토 우선순위'를 도출하는 훌륭한 의사결정 지원(Decision Support) 시스템**입니다. 

### 👍 강점 (Strengths)
*   **체계적인 모듈화 및 파이프라인 구성**: `src` 폴더 내에 `api`(데이터 수집), `features`(피처 엔지니어링), `models`(모델링), `reports`(평가 및 리포트), `viz`(시각화)로 역할이 매우 깔끔하게 분리되어 있습니다.
*   **단일 진입점(Single Entry Point)**: `run_final_pipeline.py`를 통해 전체 파이프라인을 일관되게 실행할 수 있어 재현성(Reproducibility)이 뛰어납니다.
*   **문서화 및 형상 관리**: `프로젝트_코드_구조_가이드.md`, `피드백_반영_요약.md` 등 문서화 수준이 실무 데이터 사이언스 프로젝트 상위 5% 이내에 들 정도로 매우 상세하고 체계적입니다.

### 💡 개선점 (Areas for Improvement)
*   **테스트 커버리지 확장 필요**: 핵심 산식(`compute_policy_risk_score`, `assign_policy_risk_label`)에 대한 1차 `pytest` 테스트는 추가되었습니다. 다음 단계에서는 좌표 검증, 위험등급 경계값, Streamlit 정책 가중치 재계산까지 테스트 범위를 넓히면 유지보수성이 더 좋아집니다.
*   **리포트 파편화**: `outputs/reports/`에 생성되는 CSV 파일이 40여 개에 달합니다. 중간 검증용으로는 훌륭하지만, 최종 이해관계자를 위해 이를 요약해주는 메타 리포트(HTML 또는 대시보드 형태)가 있으면 더 좋겠습니다.

## 2. 머신러닝 방법론 및 데이터 파이프라인 평가

가장 인상적인 부분은 **머신러닝의 맹점(과적합, 인과관계 오류)을 정책적 도메인 지식으로 훌륭하게 방어**하고 있다는 점입니다.

### 👍 강점 (Strengths)
*   **베이스라인 모델과의 철저한 비교**: 단순 절대값 예측이 아닌 '전년도 유지(Naive Baseline)'와 '변화량(Delta) 타겟'을 비교하여 모델의 실제 기여도를 객관적으로 증명한 점은 매우 훌륭한 접근입니다.
*   **시간 기반 교차 검증 (Time-based CV)**: Expanding Window 방식을 도입하여 미래 시점을 예측하는 시계열 특성을 올바르게 검증하고 있습니다.
*   **도메인 지식을 반영한 시나리오 분리**: 과거 폐교 데이터를 학습한 EDSS 모델의 결과를 '절대적인 폐교 확률'이 아닌 '유사도 백분위'로 해석하고, 출생 코호트 기반 장기 시나리오(낙관/기준/비관)를 별도로 두어 불확실성을 관리한 점이 돋보입니다.
*   **교육공백 보호군 분리**: 단순히 학생 수가 적고 위험점수가 높다고 통폐합으로 몰아가는 것이 아니라, '고립도'를 활용해 통폐합 시 교육공백이 우려되는 학교를 보호군으로 분리한 로직은 실제 정책 반영 시 가장 강력한 설득력을 가집니다.

### 💡 개선점 (Areas for Improvement)
*   **학습-추론 피처 정합성의 남은 한계**: 분류모델은 현재 서빙에서 안정적으로 확보 가능한 10개 피처 중심으로 축소되어 결측 대체 문제를 줄였습니다. 다만 학교 고립도/대체학교 접근성처럼 현재 좌표 기반으로만 계산 가능한 피처는 과거 EDSS 검증 모델에 직접 넣기 어렵습니다.
*   **앙상블/하이퍼파라미터 튜닝 기록**: Random Forest와 HistGradientBoosting을 사용 중이나, 최적의 파라미터를 찾는 과정(GridSearchCV / RandomizedSearchCV)에 대한 파이프라인 내 명시적 기록이나 코드가 다소 약해 보입니다.

## 3. 시각화 및 UX (User Experience) 평가

최종 산출물인 HTML 지도(`final_national_interactive_school_risk_scenario.html`)는 복잡한 데이터를 정책 입안자가 쉽게 이해할 수 있도록 잘 설계되었습니다.

### 👍 강점 (Strengths)
*   **지연 로딩(Lazy Loading) 최적화**: 전국 데이터를 한 번에 렌더링하지 않고 지역별 JS 파일로 분리하여 지도 초기 로딩 속도를 최적화한 점은 웹 프론트엔드 관점에서도 우수합니다.
*   **다차원적 데이터 표현**: '시군구 단위 감소 버블'과 '학교별 점'을 자동 전환하고, 팝업 내에서 '위험등급 카드', '학생수 요약', '점수 기여 Chip' 등 데이터를 계층적으로 잘 보여주고 있습니다.
*   **UI를 통한 시뮬레이션 지원**: 최종 HTML에서는 위험 피처를 켜고 끄는 방식으로 정책 조건 변화를 확인할 수 있고, 별도 Streamlit 앱(`src/viz/streamlit_policy_simulator.py`)에서는 가중치 자체를 슬라이더로 조절하는 what-if 실험도 가능합니다.

## 4. 종합 요약 (Conclusion)

본 프로젝트는 **단순한 데이터 분석이나 캐글(Kaggle) 스타일의 성능 지표(R², F1 Score) 올리기에 매몰되지 않고, "실제 정책에서 어떻게 쓰일 것인가?"를 치열하게 고민한 뛰어난 결과물**입니다. 

특히 '폐교 확률'이라는 단정적인 용어 대신 **'통폐합 검토 우선순위'**, **'학령수요 감소압력'**과 같은 신중한 용어를 선택하고, '교육공백 보호군'을 분리한 것은 이 프로젝트의 완성도와 전문성을 보여주는 가장 큰 장점입니다.
