# Project Directory Structure (docs/project_structure.md)

본 문서는 대외 공개용 리포지토리의 최종 디렉토리 구조 및 각 하위 폴더별 역할군을 정리하고, 보안 및 용량 문제로 GitHub 업로드 대상에서 배제 처리한 항목을 명시합니다.

---

## 1. 리포지토리 디렉토리 트리 구조

```text
v5_result/
├─ README.md                  # 대외 공개용 리포지토리 메인 설명서
├─ requirements.txt           # 파이썬 종속 패키지 리스트
├─ .gitignore                 # 릴리즈 제외 대상 정의서
├─ .env.example               # 로컬 구동용 환경변수 템플릿
├─ LICENSE                    # 오픈소스 라이선스 파일 (MIT)
├─ src/                       # 핵심 소스코드 디렉토리
│  ├─ README.md               # 소스 모듈별 역할 정의서
│  ├─ api/                    # 외부 인구 및 공간 데이터 수집 API 모듈
│  ├─ common/                 # 설정 로드 및 공간 거리 연산 유틸리티
│  ├─ features/               # R-stage 단계별 피처 엔지니어링 모듈
│  ├─ pipeline/               # 학습 및 검증 파이프라인 엔트리
│  └─ code/                   # 세부 하이퍼파라미터 튜닝 및 웹 익스포터
├─ data/                      # 데이터 디렉토리
│  ├─ README.md               # 데이터 출처 및 스키마 설명서
│  └─ sample/                 # 재현성 보증용 콤팩트 모의 샘플 데이터 CSV
├─ public/                    # 대시보드 데모 및 웹 데이터 디렉토리
│  ├─ README.md               # 웹 데스크탑 시뮬레이터 및 데이터 설명서
│  └─ data/                   # 웹 구동용 핵심 시나리오 데이터 폴더 (scenario_v5_final/)
├─ outputs/                   # 성능 지표 및 시각화 디렉토리
│  ├─ README.md               # 최종 결과 지표 명세서
│  ├─ figures/                # 보고서 삽입용 최종 시각화 PNG 이미지 12종
│  ├─ metrics/                # 모델 튜닝 최종 성능 검증 수치 CSV
│  └─ scenario_summary/       # 2030 시나리오 시계열 요약 정보
├─ docs/                      # 개발 기술 문서 디렉토리 (현재 폴더)
├─ deliverables/              # 최종 제출용 결과물 보관소
│  ├─ README.md               # 제출 문서 위치 안내서
│  ├─ report/                 # HWP 제출 보고서용 마크다운 원고 및 삽입 이미지
│  └─ paper/                  # A4 2페이지 분량이 보강된 학술 논문형 HTML 패키지
├─ assets/                    # 미디어 및 원천 리소스 저장소
│  ├─ README.md               # 소스 자산 정보
│  └─ diagram_sources/        # 차트 작성을 위한 원천 SVG 다이어그램 리소스
└─ release_audit/             # 릴리즈 전 무결성 감사 보고서 폴더
```

## 2. 각 주요 디렉토리의 역할

* **`src/`**: 데이터 연동 및 정제, 기계학습 회귀 모델의 훈련, 평가, 2030 미래 시뮬레이션의 핵심 Python 로직을 수용합니다.
* **`data/`**: 개인정보 및 위경도 절대 좌표 등 민감 raw 데이터를 배제하고, 모델 데이터 포맷에 맞춘 샘플 데이터만 탑재하여 경량화된 분석 형태를 제공합니다.
* **`public/`**: 지도 마커와 학교별 상세 진단 패널을 시각적으로 구현하여 우선점검 참고 흐름을 증명하는 프론트엔드 정적 웹 데모입니다.
* **`outputs/`**: 모델 튜닝의 과학적 근거가 되는 성능 통계치와 시나리오 요약본 및 검증된 figures를 영구 집약 보존합니다.
* **`deliverables/`**: 논문 HTML 및 보고서 원고 등 실행 코드 외부의 최종 릴리즈 제출 양식만을 관리합니다.
* **`assets/`**: 보고서와 설명서 내에 사용되는 아키텍처 다이어그램 등을 수동 편집하기 위해 보존해야 하는 원천 SVG 파일들을 보관합니다.

## 3. GitHub 업로드 제외 항목 (배제 정책)

로컬의 다음 폴더/파일들은 보안 자격 증명 유출 및 용량 제한 방지를 위해 저장소 업로드에서 제외됩니다.
* **`_archive_before_github/`**: 개발 히스토리 및 중간 실험 데이터 백업을 담은 15GB 규모의 로컬 아카이브 폴더
* **`data/raw/` 및 `data/interim/`**: 학교 주소 원천, 통계청 인구 원본 엑셀 등 대용량 데이터
* **`.env`**: 카카오 및 통계청 API 연동을 위해 사용되는 로컬 비밀 키 파일
* **`__pycache__/` 및 `*.pyc`**: 파이썬 인터프리터 런타임 캐시
