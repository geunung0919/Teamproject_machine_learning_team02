# 프로젝트 문제정리 및 해결 과정

## 1. 초기 문제: 장기 학교 데이터에는 좌표가 없었다

처음에는 장기 시계열이 있는 학교 데이터를 확보하는 것이 핵심이었다.  
학령인구 감소와 폐교 위험은 한두 해 데이터로는 검증이 어렵기 때문에 10년 이상 학교별 학생수 변화가 필요했다.

하지만 초기에 사용한 장기 학교 데이터는 다음 문제가 있었다.

- 학교별 학생수와 상태는 장기간 존재
- 학교명, 학교급, 연도, 학생수는 분석 가능
- 그러나 위도/경도 좌표가 없거나 매우 부족
- 좌표가 없어서 학교 고립도, 대체학교 거리, 지도 시각화를 계산하기 어려움

이 상태에서는 모델이 학생수 변화만 볼 수 있고, 프로젝트 차별점인 “학교 고립도”를 제대로 반영할 수 없었다.

## 2. 첫 번째 해결 시도: 현재 학교 API 좌표와 장기 데이터 매칭

현재 운영 중인 학교 API에는 학교 좌표가 있었다.  
그래서 장기 학교 데이터의 학교명과 현재 학교 API의 학교명을 매칭해서 좌표를 붙였다.

이 방법으로 현존 학교 상당수의 좌표를 확보할 수 있었다.

한계:

- 현재 학교 API에는 이미 폐교된 학교가 없음
- 과거 폐교 학교는 좌표가 거의 복구되지 않음
- 학교명 변경, 행정구역 변경, 분교명 차이 때문에 매칭이 불안정
- 폐교 위치를 모르니 “실제 폐교가 어떤 공간 조건에서 발생했는지” 검증하기 어려움

즉, 현존 학교 지도는 만들 수 있었지만 폐교 이력 기반 모델의 신뢰성이 부족했다.

## 3. 두 번째 문제: 폐교 좌표가 꼭 필요했다

폐교 좌표가 필요한 이유는 다음과 같다.

- 실제 폐교 학교의 위치 패턴을 확인해야 함
- 주변 대체학교 수, 가장 가까운 학교 거리, 고립도 계산이 필요함
- 폐교 모델이 단순 학생수만 보고 판단하는지, 공간 조건까지 보는지 확인해야 함

초기 폐교 좌표 보유율은 매우 낮았다.

- 폐교 행: 861개
- 초기 폐교 좌표 있음: 19개
- 초기 폐교 좌표 확보율: 약 2.21%

이 수준에서는 폐교 공간 분석이 사실상 불가능했다.

## 4. 해결: 주소 기반 지오코딩과 수동 주소 보정

폐교 데이터에는 좌표는 없었지만 주소는 거의 모두 있었다.  
따라서 주소를 기반으로 좌표를 다시 만들었다.

사용한 방법:

- VWorld 주소 좌표 변환 API
- Naver 지오코딩 일부 시도
- Kakao Local API 연결 시도
- 주소 띄어쓰기 정제
- 옛 지번/도로명 주소 보정
- 수동으로 다시 만든 폐교 주소 파일 반영

대표 정제 예시:

- `연주리451` -> `연주리 451`
- `두창리복분로 29` -> `두창리 복분로 29`
- `전남` -> `전라남도`
- `번지`, 괄호 설명 제거

최종 좌표 복구 결과:

- 전체 학교-연도 행: 213,680개
- 전체 좌표 확보: 211,610개
- 전체 좌표 확보율: 99.03%
- 폐교 행: 861개
- 폐교 좌표 확보: 797개
- 폐교 좌표 확보율: 92.57%
- 폐교 좌표 미확보: 64개

이 과정을 통해 학교 고립도와 대체학교 접근성 계산이 가능한 수준까지 올라갔다.

## 5. 모델링 문제: 폐교 라벨 정의가 불안정했다

초기에는 다음 연도에 학교키가 사라지면 폐교로 보는 방식도 사용했다.

문제:

- 실제 폐교가 아니라 학교명 변경, 행정구역 변경, 데이터 누락일 수 있음
- 학생수가 많은 대형 학교도 위험 상위에 등장
- 모델이 실제 폐교가 아니라 데이터 연결 오류를 학습할 가능성이 있음

따라서 이 방식은 폐기했다.

## 6. 1년 뒤 폐교 이진분류의 한계

다음으로 더 엄격하게 `1년 뒤 실제 폐(원)교 상태 전환`만 양성으로 정의했다.

장점:

- 실제 폐교에 가까운 라벨
- 잘못된 학교키 누락 문제 감소

한계:

- 폐교가 너무 희소함
- 검증 양성 비율이 약 0.2% 수준
- F1과 Precision이 낮게 나옴

성능 예시:

- HistGB F1 약 0.164
- PR-AUC 약 0.094

이 결과는 모델이 완전히 실패했다기보다는, “전체 학교 중 1년 뒤 폐교”라는 문제가 너무 희소하고 정책적으로도 좁다는 것을 보여줬다.

## 7. 최종 해결: 통폐합 검토 후보군 내부의 3년 이내 폐교 모델

최종적으로 문제를 다음처럼 재정의했다.

> 전체 학교 중 폐교 확정 예측이 아니라, 통폐합 검토 후보군 안에서 3년 이내 폐교 위험이 높은 학교를 찾는다.

후보군 기준:

- 기존 학교
- 학생수 0명/결측 제외
- 초등 80명 이하
- 중학교/고등학교 330명 이하
- 또는 분교/분교장

타깃:

- 현재 연도 기준 3년 이내 폐(원)교 상태가 되면 1

학습/검증:

- 학습: 2008~2018
- 검증: 2019~2022
- 2023~2025는 3년 이내 결과가 완전히 관측되지 않으므로 검증 제외

최종 성능:

| 모델 | Precision | Recall | F1 | ROC-AUC | PR-AUC |
|---|---:|---:|---:|---:|---:|
| Logistic Regression | 0.183 | 0.395 | 0.250 | 0.942 | 0.154 |
| HistGB | 0.292 | 0.663 | 0.406 | 0.972 | 0.344 |

Top-K, HistGB:

- Top 10: 실제 폐교 7개 포함
- Top 50: 실제 폐교 23개 포함
- Top 100: 실제 폐교 45개 포함
- Top 500: 실제 폐교 150개 포함
- Top 1000: 실제 폐교 228개 포함

이 모델은 단순 이진 정확도보다 “검토 우선순위” 관점에서 해석하는 것이 적절하다.

## 8. 학교 고립도 해석 문제

초기에는 학교 고립도를 위험 점수에 넣으려고 했다.  
하지만 검토 중 다음 사실을 확인했다.

- 고립도가 낮음: 주변 대체학교가 많아 통폐합 가능성이 큼
- 고립도가 높음: 폐교하면 교육공백이 커져 오히려 폐교하기 어려움

따라서 고립도는 폐교 위험을 높이는 피처가 아니라, **교육공백 보호 신호**로 분리했다.

최종 분류:

- 통폐합 검토 후보: 저학생수 + 대체학교 접근성 있음 + 모델 위험 높음
- 교육공백 보호대상: 저학생수이지만 고립도 높고 대체학교 부족

이렇게 해서 “폐교 가능성”과 “폐교하면 안 되는 학교”를 구분했다.

## 9. 왜 모델 성능이 이렇게 나왔는가

성능이 처음부터 높게 나오지 않은 이유는 다음과 같다.

1. 폐교는 매우 희소한 사건이다.
   - 전체 학교 중 폐교는 극히 일부라 일반 분류 지표가 불리하다.

2. 폐교는 단순 데이터 패턴이 아니라 정책 결정이다.
   - 학생수, 위치, 대체학교뿐 아니라 지역 민원, 교육청 정책, 학교 역사도 영향을 준다.

3. 학교명/행정구역/주소 변경이 많다.
   - 장기 패널에서 같은 학교를 안정적으로 추적하는 것이 어렵다.

4. 고립도는 폐교 위험과 반대 방향으로 작동할 수 있다.
   - 고립 학교는 학생수가 적어도 폐교하기 어렵다.

5. 전체 학교 대상 모델은 문제 정의가 넓다.
   - 후보군 내부 모델로 바꾸자 성능과 해석력이 개선되었다.

## 10. 최종 발표 메시지

이 프로젝트는 “폐교를 정확히 예언하는 모델”이 아니다.

보다 정확한 설명은 다음과 같다.

> 장기 학교 데이터, 좌표 복구, 학교 고립도, 상권 피처를 이용해 통폐합 검토 후보군의 우선순위를 산정하고, 동시에 교육공백 보호대상을 분리하는 모델이다.

차별점:

- 장기 학교-연도 데이터 사용
- 폐교 좌표를 주소 기반으로 복구
- 학교 고립도를 단순 위험이 아니라 교육공백 보호 신호로 해석
- Top-K 기반으로 정책 검토 우선순위 제시
- 최종 지도에서 후보 학교를 공간적으로 확인 가능
