# 천안/아산 미래 학생 수 예측 및 학교 통폐합 위험군 분류 프로젝트 기획서

## 1. 프로젝트 개요

천안/아산 지역의 학교별 미래 학생 수를 머신러닝 회귀모델로 예측하고, 예측된 학생 수와 학교 고립도, 상권 변화, 주거 특성, 학령인구 변화 등을 결합하여 학교 통폐합 고위험군을 분류한다.

이후 Folium 라이브러리를 활용해 학교별 예측 학생 수와 통폐합 위험도를 지도 위에 시각화한다.

## 2. 프로젝트 목표

### 1차 목표: 머신러닝 모델 구축

- 학교별 또는 읍면동-학교급별 미래 학생 수 예측
- 예측 결과를 기반으로 통폐합 위험도 산정
- 고위험/중위험/저위험 학교 분류

### 2차 목표: Folium 지도 시각화

- 학교 위치 지도 표시
- 예측 학생 수에 따른 마커 크기/색상 표현
- 통폐합 위험군 색상 구분
- 학교 주변 상권/고립도 피처 지도 표현

## 3. 문제 정의

### 회귀 문제

```text
입력:
과거 학생 수, 학급 수, 학령인구, 주거 특성, 상권 특성, 학교 고립도

출력:
n년 후 학교별 학생 수
```

예측 대상 예시:

```text
target_student_count_next_1y
target_student_count_next_3y
target_student_count_next_5y
```

### 분류 문제

```text
입력:
예측 학생 수, 학생 수 감소율, 학교 고립도, 대체학교 접근성, 상권/주거/인구 피처

출력:
통폐합 위험등급
```

분류 등급 예시:

```text
저위험
중위험
통폐합 가능 고위험
교육공백 우려 고위험
```

## 4. 핵심 차별화 포인트

기존 연구는 주로 학생 수, 학령인구, 지역소멸지표 중심으로 학교 통폐합 위험을 분석하는 경우가 많다.

본 프로젝트는 다음 두 가지 피처를 추가하여 차별화한다.

### 4.1 상권 데이터 활용

학교 주변 상권은 지역 생활권의 활력과 가족 단위 정주 가능성을 반영한다.

활용 가능한 피처:

```text
학교 반경 500m/1km 내 전체 상가 수
학교 반경 1km 내 학원/교육서비스 수
키즈 관련 업종 수
실버 관련 업종 수
음식점/의료/생활서비스 밀도
키즈-실버 상권 비율
```

### 4.2 학교 고립도 활용

학생 수가 적다고 무조건 통폐합 대상이 되는 것은 아니다. 주변 대체학교가 멀면 통폐합 시 교육 접근성 문제가 발생한다.

활용 가능한 피처:

```text
가장 가까운 같은 학교급 학교까지 거리
반경 2km 내 같은 학교급 학교 수
반경 5km 내 전체 학교 수
가장 가까운 대규모 학교까지 거리
학구도 기반 대체학교 접근성
```


## 8. 모델링 전략

### 8.1 학생 수 예측 회귀모델

우선 비교할 모델:

```text
Linear Regression
Ridge / Lasso
Random Forest Regressor
Gradient Boosting Regressor
XGBoost 또는 LightGBM
```

초기 기준 모델은 단순해야 한다.

```text
baseline_1: 전년도 학생 수 유지
baseline_2: 최근 3년 평균 증감률 적용
baseline_3: 읍면동 학령인구 증감률 적용
```

머신러닝 모델은 이 baseline보다 성능이 좋아야 의미가 있다.

### 8.2 통폐합 위험 분류

실제 통폐합 라벨이 충분한 경우:

```text
Logistic Regression
Random Forest Classifier
Gradient Boosting Classifier
XGBoost / LightGBM Classifier
```

라벨이 부족한 경우:

```text
규칙 기반 위험점수
KMeans / Gaussian Mixture Clustering
Isolation Forest
```

초기에는 규칙 기반 점수로 위험등급을 만들고, 이후 실제 폐교/통폐합 사례를 확보하면 지도학습으로 전환한다.

## 9. 위험등급 설계

단순히 학생 수가 적은 학교를 고위험으로 두지 않는다.

### 9.1 통폐합 가능 고위험군

```text
예측 학생 수 낮음
학생 수 감소율 높음
주변 같은 학교급 학교 많음
가까운 대체학교 존재
상권/주거 기반 약화
```

### 9.2 교육공백 우려 고위험군

```text
예측 학생 수 낮음
학생 수 감소율 높음
주변 같은 학교급 학교 적음
대체학교까지 거리 멂
학구도상 고립도 높음
```

이 구분이 프로젝트의 핵심 차별점이다. 통폐합 대상과 유지 필요 대상을 분리해서 해석할 수 있다.

## 10. 평가 지표

### 회귀모델 평가

```text
MAE
RMSE
MAPE
R2 Score
```

학생 수 예측에서는 해석이 쉬운 MAE를 주요 지표로 둔다.

### 분류모델 평가

```text
Accuracy
Precision
Recall
F1-score
Confusion Matrix
```

고위험군을 놓치지 않는 것이 중요하므로 Recall과 F1-score를 중요하게 본다.

## 12. 개발 단계

### 1단계: 데이터 수집

- 학교알리미에서 학교별 학생 수/학급 수 수집
- NEIS에서 학교 기본정보 수집
- KOSIS에서 읍면동별 학령인구 수집
- 학교 주소를 좌표로 변환

### 2단계: 데이터 정제 및 병합

- 학교코드 기준 병합
- 행정동/법정동 코드 정리
- 좌표 오류 검수
- 연도별 패널 데이터 생성

### 3단계: 피처 엔지니어링

- 학생 수 증감률
- 학령인구 증감률
- 학교 고립도
- 상권 밀도
- 주거 특성

### 4단계: 회귀모델 구축

- baseline 모델 생성
- 여러 회귀모델 비교
- 성능 평가
- 주요 변수 중요도 확인

### 5단계: 위험등급 분류

- 예측 학생 수 기반 위험점수 생성
- 통폐합 가능 고위험/교육공백 우려 고위험 분리
- 실제 라벨 확보 시 분류모델 학습

### 6단계: Folium 시각화

- 학교별 예측 결과 지도화
- 위험군 레이어 생성
- 상권/고립도 팝업 추가
- HTML 지도 저장

## 16. 현재 가장 중요한 의사결정

가장 먼저 정해야 할 것은 예측 단위이다.

권장안:

```text
1순위: 학교-연도 단위 예측
2순위: 읍면동-학교급-연도 단위 예측
```

학교별 과거 학생 수를 충분히 확보할 수 있으면 학교 단위 모델로 간다. 확보가 어렵다면 읍면동-학교급 단위로 먼저 모델을 만들고, 이후 학교별 공간 피처를 붙여 위험등급을 세분화한다.
